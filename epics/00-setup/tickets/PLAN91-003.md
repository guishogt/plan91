### Branch Strategy

**Main Branches**:
- `main` - Production-ready code
- `develop` - Integration branch for features

**Supporting Branches**:
- `epic/{epic-id}` - One branch per epic
- `ticket/{ticket-id}` - One branch per ticket (optional)

**Workflow**:
1. Create epic branch from develop
2. Work on tickets in epic branch
3. Merge epic back to develop when complete
4. Merge develop to main for releases
